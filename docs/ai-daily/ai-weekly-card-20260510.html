4 Claude Code /loop · 把 Agent 变成终端 cron AI 原语
痛点：监控部署/轮询 PR/定时拉日志——过去要么裸写 bash 脚本，要么开 Zapier / n8n。/loop 把"轮询 + 异常判断"揉成一条 AI 原语，Claude 既负责巡检也负责判断什么情况该告警。
# /loop 三种用法 /loop 5m /check-deploy # 每 5 分钟跑一次 /loop /watch-pr # 让模型自行判断节奏 # 降本套餐：长跑任务组合使用 /effort low # 简单巡检降 token /focus # 压掉 TUI 噪音 /radio # Claude FM lo-fi 陪跑
兼容
所有订阅计划 · Sonnet 4.6 / Haiku 4.5 均可
高频警告
< 5 分钟轮询建议 Pro/Max + effort low

5 Claude Code /ultrareview · 云端多 Agent 联合代码审查 云端单独计费
痛点：一个人盯不住 PR 的"安全 × 性能 × 业务逻辑"三条线，找三位同事要排期到明天。/ultrareview 在云端同时起多个专精 Agent，返回一份结构化审查报告（和本地速率上限独立）。
# 两种触发方式 /ultrareview # 审当前分支 /ultrareview 12345 # 审指定 PR # 组合 /loop：每晚自动审 main 分支 /loop 24h /ultrareview main
过去
一次 PR 至少 30 分钟找人 + 1-2 天等 review
现在
10 分钟拿到"三方评审视角"初稿
无远程也能用
本地分支自动打包上云

6 Native Git Worktrees · 多 Agent 并行真正互不打架 自动清理
痛点：多个 Claude Code 窗口在同目录跑 → 改文件互相覆盖、git 状态混乱。Native Worktrees 让每个 sub-agent 用独立临时 worktree，PR squash-merge 后自动清理，不留孤儿分支。
# baseRef 配置 worktree.baseRef: fresh # 新功能：从最新 main 创建（推荐） worktree.baseRef: head # Bugfix：从当前 commit 创建 # 委派 sub-agent 时声明隔离 { "isolation": "worktree" } # Claude 会自动克隆 worktree + 生成带上下文的 CLAUDE.md
新功能开发
baseRef: fresh
Bugfix / 调试
baseRef: head

iii. Methods & Prompts 3 items
🧠 方法论与 Prompt
3 套最有 ROI 的一次性投入
7 Opus 4.7 Effort 级别 + 1H 缓存 · 长会话降本 3-4 倍 73% 省钱
1M 上下文时代，成本不再由模型聪不聪明决定，而是"你有没有把它塞进缓存"。Opus 4.7 引入 /effort 级别 + 1 小时 TTL prompt caching，实测长会话降本 3-4×（某 2 万行项目从 $120 降至 $32）。
# 1) 开启 1 小时缓存（默认 5 分钟） export ENABLE_PROMPT_CACHING_1H=1 # 2) 按任务类型切 effort /effort low # 简单查询 / 高频 loop /effort medium # 日常调试（默认推荐） /effort high # 安全分析 / 多文件重构 / 架构推理 /effort xhigh # 复杂深度推理（耗 token） /effort auto # 仅 Max 订阅可用 # 3) 启动长会话 claude --project ./myapp --context-window 1m
新手最大坑
effort 和任务不匹配：简单任务开 xhigh 烧钱、复杂任务留 low 出 bug
稳妥起手
先用 medium 跑通基线，再按实际效果调档

8 Meta-Prompting · 让 AI 给 AI 写 Prompt 的脚手架 Level 6 模板
与其人肉调 prompt，不如让 Meta Agent 帮你生成/评估/迭代。Anthropic 把这套模式正式纳入 Plugin Marketplace（Skill 名 Template Meta-Prompt Creator），产出"Level 6"标准化 Prompt。
# Meta Prompt 模板（直接粘到 Claude / ChatGPT） 你是一个专业的 Prompt 工程师。请为以下任务生成 Level 6 Meta Prompt： # 任务描述 [一句话说清楚你希望 AI 完成什么] # 使用场景 [AI 将在什么工作流中被调用？输入来自哪？输出给谁看？] # 成功标准 [3-5 个可验证的 checklist] # 已知失败案例 [过去试过输出不满意的 prompt / 示例] 请输出： 1) Role 定义（≤50 字） 2) Task 拆解（分 3-5 步） 3) Few-shot 示例（≥2 组 input/output） 4) 自我检查清单 5) 输出格式（JSON / Markdown / 纯文本）
关键诀窍
把"失败案例"喂回 meta prompt，3 轮迭代后稳定度 +60%
实战
微信公众号 → 小红书文案 prompt：Meta Agent 输出 300 字结构化 prompt + 2 组 few-shot，直接复用 20+ 次
