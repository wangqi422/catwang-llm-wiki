- Manus / OpenManus
- OpenClaw（龙虾）
- Claude Coworker / Skills
- Coze / 扣子

- AutoGPT类、LangChain类自建方案等

MuleRun的核心优势（目前阶段最突出的点）

1. 真正持久在线 + 独立虚拟机：任务可以“今晚布置，明早看结果”，非常适合需要长时间运行的任务（游戏挂机、数据监控、建模渲染等），这是绝大多数竞品做不到的。
2. 最像“淘宝”的Agent市场：供给丰富（已超200+个高质量Agent），创作者分成清晰，用户找工具像逛淘宝一样方便。
3. 0门槛 + 专业级输出：非技术人员也能用，不需要自己写代码、配环境，就能调用别人封装好的复杂能力（Blender建模、游戏全自动日常、投研报告等）。
4. 创作者友好：有现金激励、流量扶持、上架补贴，吸引了很多领域专家把know-how变成可售卖的“数字劳动力”，供给质量相对更高。
5. 自进化 + 群体智慧：个人层面持续记住你，群体层面可以共享/匹配高权重的最佳实践Agent。

MuleRun的主要缺点/短板（2026年3月阶段）

1. 必须联网 + 云端运行：不像本地Agent那样关机也能思考，也不像本地那样完全掌控数据（虽然独立虚拟机隐私隔离做得不错，但仍然是云）。
2. 成本：按积分/次数/时长付费，复杂任务或长时间运行成本会明显高于本地方案或简单聊天（虽然有免费额度）。
3. 平台仍年轻：虽然用户增长很快（上线几个月就几十万用户），但Agent质量仍然参差，偶尔有Bug或不稳定，售后响应有时跟不上。
4. 对超级个性化深度记忆稍弱于纯本地方案（如OpenClaw），因为云端虚拟机无法直接操作你的真实本地文件/设备。
5. 创作门槛其实不低：虽然宣称0代码，但做出真正好用、能赚钱的Agent仍需要懂流程 + 懂prompt工程 + 懂Claude Code等，普通用户很难做出爆款。
