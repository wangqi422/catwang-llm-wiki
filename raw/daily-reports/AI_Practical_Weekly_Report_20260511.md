## ⚡ 高效工作流 (Workflows & Hacks)

### 4. Midjourney V7 完整工作流：Draft Mode + Omni-Reference

- **痛点场景**：AI 出图迭代慢（每次等 1 分钟）、角色跨镜头变脸、风格不一致
- **AI 解决路径**：MJ V7 的 **Draft Mode** 让概念迭代速度提升 10 倍（成本降低 50%），**Omni-Reference** 实现角色一致性锁定：
  1. **Draft Mode 快速试错**：用 10 秒级低分辨率生成快速验证构图/风格
  2. **Omni-Reference 锁定角色**：上传 1-4 张参考图，V7 自动提取角色特征
  3. **正式渲染**：确认后用 Fast/Relax 模式出高清图
- **预期效果**：过去"出 10 张图挑 1 张" → 现在"Draft 30 张选 3 张精修"，整体效率提升 3-5 倍
- **Prompt 模板**：
```
[主体描述] + [环境/光线/镜头] + [风格关键词],
--ar 16:9 --sref [风格图URL] --cref [角色图URL] --stylize 300 --v 7
```

*Source: @midjourney*

### 5. Claude Code Git Worktrees：多 Agent 并行真正互不打架

- **痛点场景**：多个 Claude Code 窗口在同目录跑 → 改文件互相覆盖、git 状态混乱、孤儿分支堆积
- **AI 解决路径**：Claude Code v2.1.13x 引入 **Native Git Worktrees**——每个 sub-agent 用独立临时 worktree，PR squash-merge 后自动清理：
  1. 配置 `worktree.baseRef: fresh`（新功能从最新 main 创建）或 `head`（Bugfix 从当前 commit）
  2. 委派 sub-agent 时声明 `"isolation": "worktree"`
  3. Claude 自动克隆 worktree + 生成带上下文的 CLAUDE.md
- **预期效果**：多 Agent 并行开发从" chaos "变成"井然有序"，分支管理零 overhead
- **适用场景**：大型重构任务拆分给多个 sub-agent / 同时处理多个独立 feature

*Source: @AnthropicAI*

### 7. Veo 3.1 视频生成工作流：从"能生成"到"能商用"

- **核心理念**：Google Veo 3.1 带来 **9:16 原生竖屏** + **4K 60fps** + **多参考图角色一致性**，AI 视频从"玩具"进化为"可商用素材"。
- **Prompt 模板（3段式）**：
```
[主体: 谁/做什么] +
[环境: 地点/时间/氛围] +
[镜头: 运镜方式/焦距/角度] +
[光线: 光源类型/质感]
```

  - 物理模拟更真实（流体/人体动作）
  - 面部表情更细腻（情绪转变连贯）
  - 音频同步增强（原生音效匹配画面）
